5.3 如何创建systemd 自启动服务

  想开机自启动程序，可以在/etc/profile.d/qt-eglfs.sh这个文件里添加启动程序的指令。也可以自己创建新的系统服务，让系统启动起来时自动执行。如果觉得麻烦，可以参考5.5小节，不使用systemd创建自启动程序。

  我们先创建一个要启动的脚本

vi auto_run_script.sh

  在脚本里添加以下内容，脚本内容是打印信息到串口。

#!/bin/bash
echo “Hello World!”> /dev/ttySTM0

  赋予脚本可执行权限，执行下面的指令

chmod +x auto_run_script.sh

  我们进入/etc/systemd/system目录下，创建一个自启动服务

auto_run_script.service。
cd /etc/systemd/system
vi auto_run_script.service

  在auto_run_script.service里添加以下内容

[Unit]
Description=Run a Custom Script at Startup
After=default.target

[Service]
ExecStart=/home/root/auto_run_script.sh

[Install]
WantedBy=default.target

  然后我们使用更新 systemd 配置文件，并建立符号链接关系

systemctl daemon-reload
systemctl enable auto_run_script.service

  再执行重启命令reboot

sync;reboot

  重启时看到打印该服务的描述信息及打印脚本的信息，表明已经执行了自启动脚本。
